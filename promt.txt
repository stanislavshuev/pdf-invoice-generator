Напиши Python-скрипт `pdf_invoice_generator.py` для генерации PDF по данным из CSV/JSON и HTML-шаблону.

Функциональность:
1) Структура директорий (относительно скрипта):
- `/data` — входные файлы данных (`.csv`, `.json`)
- `/templates` — HTML-шаблоны (`.html`, `.htm`)
- `/output` — итоговые PDF

2) При запуске:
- вывести нумерованное меню доступных файлов данных из `/data`;
- вывести нумерованное меню HTML-шаблонов из `/templates`;
- дать пользователю выбрать один файл данных и один шаблон через `input`.

3) Загрузка данных:
- CSV: парсить через стандартную библиотеку `csv` (допускается `pandas`, но не обязательно);
- JSON: парсить через стандартную библиотеку `json`;
- поддержать разные кодировки CSV (минимум `utf-8-sig`, `utf-8`, `cp1251`);
- поддержать разделители CSV: `;`, `,`, `|`, tab.

4) Ключевое требование по CSV:
- корректно обрабатывать не только табличный CSV, но и полуструктурированный CSV счета (выгрузки вида банковского бланка);
- извлекать значения для полей шаблона из строк, содержащих:
  - `Счет №...` -> `invoice_id`
  - `Поставщик:` -> `supplier`
  - `Покупатель:` -> `buyer`
  - `Всего к оплате:` -> `total_amount`
  - `Основание:` -> `basis`
  - банковские реквизиты:
    - `bank_name`
    - `bik`
    - `bank_account`
    - `inn`
    - `kpp`
    - `company_name`
    - `recipient_account`
- если табличный парсинг дает нерелевантные ключи, автоматически выбирать fallback-парсинг полуструктурированного счета.

5) Выбор счета:
- показать список доступных `invoice_id` из выбранного файла;
- если найден ровно один `invoice_id`, выбирать его автоматически (без запроса);
- если найдено несколько — дать пользователю выбрать через нумерованное меню;
- если `invoice_id` не найден, дать выбрать запись по номеру.

6) Рендер шаблона:
- поддержать плейсхолдеры двух форматов:
  - `{{ field_name }}`
  - `{field_name}`
- подставлять данные выбранной записи;
- нераспознанные плейсхолдеры вида `{{ ... }}` очищать в пустую строку.

7) Генерация PDF:
- использовать `WeasyPrint`;
- обеспечить кириллицу (подключить DejaVu Sans или Roboto через CSS `@font-face`, использовать доступные системные пути шрифтов);
- сохранить PDF в `/output` с именем по `invoice_id` (безопасно нормализовать имя файла);
- после генерации автоматически открыть PDF:
  - Windows: `os.startfile`
  - macOS: `open`
  - Linux: `xdg-open`.

8) UX и надежность:
- аккуратное консольное меню с нумерацией;
- понятные сообщения об ошибках;
- создание директорий `/data`, `/templates`, `/output`, если их нет;
- обработка `KeyboardInterrupt`.

9) Критерий готовности:
- в сгенерированном PDF поля не пустые и заполняются значениями из CSV/JSON;
- для полуструктурированного CSV корректно подставляются хотя бы:
  `invoice_id`, `supplier`, `buyer`, `total_amount`, `basis` и банковские поля.
